This feature will allow an administrator to visually construct and manage the layout of various pages (Homepage, Category/Search Pages, Ad Creation Wizard) using a component-based, drag-and-drop interface from the admin panel.

Key requirements for the Page Builder:
- It should operate on a Slot and Component model. Slots are predefined areas on a page, and Components are reusable UI blocks (e.g., image gallery, price box, map) that can be placed into these slots.

- The admin should be able to define different layouts for different contexts (e.g., a specific layout for the Emlak category vs. the Vasıta category).
- The system must support a draft/publish/rollback workflow for layout changes.

**Key Technical Concepts**
-   **Hybrid Database Architecture:** The application now uses both **MongoDB** (for existing features) and **PostgreSQL** (for the new Page Builder feature).
-   **ORM and Migrations:** **SQLAlchemy** is used as the ORM for PostgreSQL, and **Alembic** manages database schema migrations.
-   **Component-Based Page Builder:** The core architectural pattern involves dynamically assembling pages from pre-defined, reusable components based on a layout definition stored in the database.
-   **Draft/Publish/Revision System:** A versioning pattern allowing for safe editing and rollback of page layouts.
-   **Transactional Backend Services:** Business logic is encapsulated in services that use transactions to ensure data integrity across multiple database operations.
-   **Zustand:** A lightweight state management library used for the complex state of the new Admin UI.
